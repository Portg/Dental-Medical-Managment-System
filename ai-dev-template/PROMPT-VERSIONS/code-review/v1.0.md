## 意图
以 intent.md Anti-Goals 为基准执行代码审查。
不只是检查代码风格——重点检查业务合规性和安全性。

## 上下文加载顺序
1. `@ai-spec/domain/intent.md`（Anti-Goals 全文）
2. `@ai-spec/domain/state-machines.yaml`
3. `@ai-spec/engine/rule-engine.md`
4. `@CLAUDE.md`
5. `@TEST-SPEC.md`
6. [待 Review 的代码文件]
7. [对应的测试文件]

## 任务描述
审查代码，输出三级问题报告

## 审查检查项

### 严重级（阻断合并）
- [ ] 金额计算是否使用浮点运算而非 bcmath？
- [ ] Controller 中是否包含业务逻辑（if/else 判断）？
- [ ] 患者敏感信息（NIN、完整姓名）是否出现在日志中？
- [ ] 状态流转是否有非法跳跃路径？
- [ ] 软删除查询是否遗漏 `whereNull('deleted_at')`？
- [ ] 折扣/退款审批流程是否被绕过？
- [ ] 病历修改是否绕过 Amendment 审批？
- [ ] 测试文件是否缺失？

### 一般级（需修复后合并）
- [ ] 翻译是否硬编码中文而非使用 `__()`？
- [ ] Blade 视图是否有内联大段 CSS/JS？
- [ ] 是否缺少权限中间件 `->middleware('can:...')`？
- [ ] 缓存操作后是否遗漏清理？
- [ ] DictItem 枚举是否硬编码而非查数据字典？

### 建议级（可后续优化）
- [ ] Service 方法是否超过 30 行？
- [ ] 是否有 3 层以上 if 嵌套？
- [ ] 是否有重复代码模式可提取？
- [ ] 复杂逻辑是否标注 `// @AiGenerated`？
