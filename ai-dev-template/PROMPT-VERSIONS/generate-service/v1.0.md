# 生效日期：2026-03-07  |  初始版本

## 变更说明
- 初始版本：基于口腔医疗管理系统 Service 层规范创建

---

## 意图
生成包含完整业务逻辑的 Service 类。
Service 是业务规则的唯一归属地——Controller 不写业务逻辑，Model 不编排流程。

## 上下文加载顺序
1. `@ai-spec/domain/intent.md`（Anti-Goals 全文）
2. `@ai-spec/domain/entities.md`（涉及的实体及关系）
3. `@ai-spec/domain/state-machines.yaml`（状态流转规则）
4. `@ai-spec/engine/rule-engine.md`（业务规则 Key）
5. `@CLAUDE.md`
6. `@TEST-SPEC.md`

## 任务描述
生成：Service 类 + 对应 PHPUnit 测试骨架

## 验收条件
- [ ] Service 放在 `app/Services/` 目录
- [ ] 使用构造函数注入依赖（Model 或其他 Service）
- [ ] 金额计算全部使用 `bcadd()`/`bcsub()`/`bcmul()`/`bcdiv()`
- [ ] 关键操作使用 `DB::transaction()` 包裹
- [ ] 状态流转调用 Model 封装的方法（如 `$queue->callPatient()`）
- [ ] 业务阈值引用 rule-engine.md 的 Rule Key，不硬编码
- [ ] 查询使用 Eloquent ORM，包含软删除处理
- [ ] 异常使用具体异常类，不抛裸 `\Exception`
- [ ] 每个公共方法有对应测试
- [ ] Anti-Goals 中每条规则有对应测试用例
- [ ] 复杂逻辑标注 `// @AiGenerated`

## 输出格式
1. 《生成决策说明》（含推理链）
2. 代码文件（Service 类）
3. 测试文件（PHPUnit，含 Anti-Goals 测试）
