# generate-api  v1.0
# 生效日期：2026-03-07  |  初始版本

## 变更说明
- 初始版本：基于口腔医疗管理系统 API v1 规范创建

---

## 意图
生成符合口腔医疗管理系统规范的 RESTful API 接口层（Controller + Route）。
不是简单的 CRUD 脚手架——需要理解业务规则（Anti-Goals）后再生成。

## 上下文加载顺序
1. `@ai-spec/domain/intent.md`（Anti-Goals + 数据敏感级别）
2. `@ai-spec/domain/entities.md`（仅涉及的实体）
3. `@ai-spec/domain/state-machines.yaml`（涉及的状态流转）
4. `@ai-spec/contracts/error-codes.md`
5. `@CLAUDE.md`
6. `@TEST-SPEC.md`（测试要求）
7. `routes/api/v1.php`（现有路由，避免冲突）

## 任务描述
生成：API Controller + Route 定义 + Request 验证 + 对应测试骨架

## 验收条件
- [ ] Controller 继承 `App\Http\Controllers\Api\V1\ApiController`
- [ ] Controller 中不含任何 if/else 业务判断（业务逻辑在 Service 层）
- [ ] 所有入参使用 `Validator::make()` + 规则数组校验
- [ ] 返回结构严格匹配 `['success' => bool, 'data' => [], 'message' => '', 'meta' => []]`
- [ ] 金额字段验证规则使用 `numeric|min:0`
- [ ] 路由定义在 `routes/api/v1.php` 中，使用 `auth:sanctum` 中间件
- [ ] 异常处理使用统一格式，不在 Controller 中 try-catch
- [ ] intent.md Anti-Goals 是否有对应测试用例？
- [ ] 复杂逻辑是否标注了 `// @AiGenerated`？
- [ ] 患者敏感信息（NIN）没有出现在日志中？

## 输出格式
1. 《生成决策说明》（含推理链，引用 Anti-Goals）
2. 代码文件（Controller / Route）
3. 测试骨架文件（Feature Test）
