## 意图
为已生成的业务代码补全测试套件，
确保 Anti-Goals 中每条规则都有对应的失败场景测试。

## 上下文加载顺序（CONTEXT-SLICE 切片 G）
1. `@ai-spec/domain/intent.md`（Anti-Goals 全文）
2. `@ai-spec/domain/state-machines.yaml`
3. `@ai-spec/engine/rule-engine.md`
4. `@TEST-SPEC.md`
5. [待测试的代码文件]

## 任务描述
生成：PHPUnit 测试文件（Feature + Unit）

## 验收条件
- [ ] 每条 Anti-Goal 都有对应的测试方法，方法名包含业务语义
- [ ] 所有非法状态跳跃都有 `$this->expectException()` 测试
- [ ] Service 层覆盖率达到 90% 以上
- [ ] 没有空测试方法或被跳过的测试
- [ ] 使用 Laravel Factory 生成测试数据
- [ ] 金额边界值覆盖：0, 0.01, 500 (审批阈值), 500.01, 99999.99
- [ ] Feature 测试使用 `$this->actingAs($user)` 模拟认证
- [ ] API 测试验证响应结构 `assertJsonStructure(['success', 'data', 'message'])`
- [ ] 测试方法命名：`it_<动词>_when_<条件>` 格式

## 输出格式
1. 测试文件（Feature / Unit）
2. Factory 文件（如果缺失）
3. 测试覆盖说明（列出每个 Anti-Goal 对应的测试方法）
