# VIBE-CHECKLIST.md — AI 输出六关验收门禁（口腔医疗管理系统）

## 【第一关】意图对齐
- [ ] AI 输出是否解决了 intent.md 里的核心业务问题？
- [ ] intent.md 中的 Anti-Goals 有没有被违反？（逐条核对）
- [ ] AI 有没有发明 intent.md 未定义的业务规则？

## 【第二关】上下文遗漏
- [ ] state-machines.yaml 中所有状态都被正确处理了？
- [ ] 命名与 CLAUDE.md / entities.md 完全一致？
- [ ] 没有多出 error-codes.md 未定义的错误码？
- [ ] DictItem 枚举值是否使用数据字典而非硬编码？

## 【第三关】AI 幻觉风险
- [ ] 没有自行发明 entities.md 里没有的字段或关系？
- [ ] 没有硬编码的魔法数字（应引用 rule-engine.md 的 Rule Key）？
- [ ] 注释描述的逻辑和代码实际实现一致？
- [ ] 没有引入项目不存在的 PHP 包或 JS 库？

## 【第四关】医疗场景专项
- [ ] 金额计算全部使用 `decimal:2` cast + `bc*()` 函数？
- [ ] 状态流转操作是否在 Model 方法中封装（非 Controller 直接修改）？
- [ ] 患者敏感信息（身份证号、病历详情）没有出现在日志中？
- [ ] 病历修改是否走合规流程（Amendment 审批）？
- [ ] 折扣 > 500 元是否触发审批流程（BR-035）？
- [ ] 退款 > 100 元是否触发审批流程？
- [ ] 软删除查询是否正确处理（`whereNull('deleted_at')`）？

## 【第五关】代码质量底线
- [ ] 没有 `dd()` / `dump()` / `var_dump()` 调试语句残留？
- [ ] 所有 TODO 注释有对应 Issue？
- [ ] 没有 `@SuppressWarnings` 或 `// @codingStandardsIgnore`？
- [ ] 没有裸 `catch (\Exception $e)` 吞掉异常？
- [ ] Blade 视图中没有内联大段 `<style>` 或 `<script>`？
- [ ] 翻译使用 `__()` / `LanguageManager.trans()` 而非硬编码中文？

## 【第六关】重构信号检测
- [ ] 这个类的职责超过一个了吗？（单一职责检查）
- [ ] 有没有超过 30 行的 Service 方法？（方法过长信号）
- [ ] 有没有超过 3 层的 if 嵌套？（复杂度过高信号）
- [ ] 有没有重复出现 3 次以上的代码模式？（重复代码信号）
- [ ] 复杂逻辑是否已标注 `// @AiGenerated` 注释？
- [ ] Scratchpad 推理链中标注为「不确定」的部分是否已标注？

## 【附加】测试覆盖验收
- [ ] 是否同步生成了测试文件？
- [ ] Anti-Goals 中每条规则是否有对应测试方法？
- [ ] 所有非法状态跳跃是否有断言测试？
