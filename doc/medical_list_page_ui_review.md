# 医疗系统 · 患者列表页 UI 规范

## 1. 设计目标（Design Goals）

- **快速定位患者**：支持多维度搜索与筛选
- **降低录入与查询认知成本**：清晰的信息层级与操作引导
- **支持高频、长期使用**：稳定的布局，减少视觉疲劳
- **符合医疗系统"专业、克制、可信"特征**：避免花哨设计，强调功能性

------

## 2. 页面信息层级（Information Hierarchy）

患者列表页必须严格划分为 **四层结构**：

```
L0 面包屑导航
L1 页面级 Header（标题 + 主操作）
L2 查询 / 筛选区
L3 数据列表区（含表头 + 数据行 / 空状态）
L4 辅助信息区（分页 / 统计）
```

⚠️ **核心原则**：任何元素 **不得跨层出现**
- 主操作按钮只能在 L1
- 筛选控件只能在 L2
- 导出功能属于页面级操作，归入 L1

------

## 3. 面包屑导航规范（L0）

### 3.1 标准结构

```
🏠 首页  /  患者中心  /  患者列表
 可点击     可点击       当前页(不可点击)
 #409EFF    #409EFF      #909399
```

### 3.2 规范要求

| 项目 | 规范 | 说明 |
|------|------|------|
| 字号 | 13-14px | 比正文略小 |
| 分隔符 | `/` 或 `>` | 颜色 #C0C4CC |
| 可点击项 | 主色 #409EFF | 有 hover 下划线效果 |
| **当前页** | **#909399 灰色** | **不可点击，无 hover 效果** |
| 首页图标 | 可选 🏠 | 16px |
| 位置 | 页面顶部，Header 上方 | 与内容区左对齐 |

### 3.3 ❌ 禁止项

| 禁止项 | 原因 |
|--------|------|
| 当前页使用蓝色/紫色 | 看起来像可点击链接，造成误导 |
| 当前页可点击 | 点击当前页无意义 |
| 面包屑过长 | 最多 4 级，超出用省略号 |

### 3.4 代码示例

```vue
<el-breadcrumb separator="/">
  <el-breadcrumb-item :to="{ path: '/' }">
    <el-icon><HomeFilled /></el-icon>首页
  </el-breadcrumb-item>
  <el-breadcrumb-item :to="{ path: '/patient' }">
    患者中心
  </el-breadcrumb-item>
  <el-breadcrumb-item>患者列表</el-breadcrumb-item>  <!-- 无 :to，不可点击 -->
</el-breadcrumb>

<style lang="scss">
// 当前页灰色
.el-breadcrumb__item:last-child .el-breadcrumb__inner {
  color: #909399;
  font-weight: normal;
  cursor: default;
  
  &:hover {
    color: #909399;  // hover 不变色
    text-decoration: none;
  }
}
</style>
```

## 4. 页面级 Header 规范（L1）

### 4.1 标准结构

```
┌─────────────────────────────────────────────────────────────┐
│  患者列表                        ╭──────╮ ╭────────────╮    │
│                                  │ 导出 │ │ + 添加患者 │    │
│                                  ╰──────╯ ╰────────────╯    │
│                                   次要      Primary         │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 规范要求

| 项目 | 规范 | 说明 |
|------|------|------|
| 页面标题 | 18-20px，font-weight: 600 | 与面包屑末级保持一致 |
| 副标题 | **禁止使用** | 面包屑已提供层级信息 |
| 主操作按钮 | Primary Button，仅 1 个 | 蓝色实心，右上角 |
| 次要操作 | Secondary Button | 如"导出"，置于主按钮左侧 |
| 按钮间距 | 12px | 多个按钮时的间隔 |
| Header 高度 | 56–64px | 包含上下内边距 |
| 背景色 | 与内容区一致 | 白色或微灰 |

### 4.3 ❌ 禁止项

| 禁止项 | 原因 |
|--------|------|
| 主操作使用文字链接 | 视觉层级不足，难以发现 |
| 标题下方加副标题 | 与面包屑信息重复 |
| 多个 Primary Button | 分散用户注意力 |
| 将"导出"放在筛选区 | 功能分类混乱 |

## 5. 查询 / 筛选区规范（L2）

### 5.1 布局原则

- **最多两行**（超出部分折叠到"高级筛选"）
- **高频条件在上**，低频条件在下或折叠
- **控件宽度统一**，视觉对齐
- **按钮组右对齐**
- **有背景色区分**

### 5.3 筛选区背景规范

| 项目 | 规范 | 说明 |
|------|------|------|
| 背景色 | **#f5f7fa** 或 **#fafafa** | 与页面背景区分 |
| 内边距 | 16px | 四周统一 |
| 圆角 | 8px | 与卡片一致 |
| 边框 | 可选，1px #ebeef5 | 或无边框仅靠背景区分 |
| 与上下元素间距 | 16-20px | 保持呼吸感 |

### 5.4 搜索框规范

| 项目 | 规范 | 说明 |
|------|------|------|
| **前置图标** | **🔍 必须有** | 帮助用户快速识别 |
| Placeholder | "搜索患者姓名/电话" | 说明可搜索的字段 |
| 宽度 | 240-280px | 比其他控件宽 |
| 圆角 | 6px | 统一规范 |

```vue
<el-input 
  v-model="searchText" 
  placeholder="搜索患者姓名/电话"
  :prefix-icon="Search"
  clearable
/>
```

### 5.5 日期范围选择器规范

**推荐**：使用单个日期范围组件，而非两个独立日期框

| 方案 | 展示 | 推荐度 |
|------|------|--------|
| 日期范围组件 | `[📅 2024-01-01 → 2024-01-31]` | ⭐⭐⭐ |
| 两个独立日期框 | `[开始日期] 至 [结束日期]` | ⭐⭐ |

```vue
<!-- 推荐：日期范围选择器 -->
<el-date-picker
  v-model="dateRange"
  type="daterange"
  range-separator="→"
  start-placeholder="开始日期"
  end-placeholder="结束日期"
  :prefix-icon="Calendar"
/>
```

### 5.6 控件规范

| 项目 | 规范 | 说明 |
|------|------|------|
| 控件高度 | 32–36px | 全局统一 |
| 输入框宽度 | 200–240px | 搜索框可更宽 |
| 下拉框宽度 | 120–160px | 根据内容调整 |
| 控件间距 | 16px 水平，12px 垂直 | 使用栅格系统 |
| **必须有标签** | 每个控件上方或左侧 | 如"来源"、"状态" |
| **圆角** | **6px** | 统一规范 |

### 5.7 按钮规范

| 按钮 | 样式 | 位置 |
|------|------|------|
| 查询/搜索 | **Primary Button** | 筛选区右侧 |
| 重置/清除 | Secondary Button | 查询按钮左侧 |
| 高级筛选 | Text Button + 展开图标 | 筛选区右下角 |

### 5.8 ❌ 禁止项

| 禁止项 | 原因 |
|--------|------|
| 筛选区无背景区分 | 层级不清晰 |
| 搜索框无图标 | 不直观 |
| 下拉框无标签 | 用户无法理解字段含义 |
| 搜索按钮非 Primary | 主操作不突出 |
| 导出放在筛选区 | 导出是页面级操作 |
| 高级筛选居中悬浮 | 位置不固定，视觉混乱 |

## 6. 按钮规范（重点）

### 6.1 按钮类型与使用场景

| 类型 | 样式 | 使用场景 | 每页数量 |
|------|------|----------|----------|
| **Primary** | 蓝色实心 | 主操作（添加、搜索、确认） | 每个区域最多 1 个 |
| **Secondary** | 灰色边框 | 次要操作（重置、取消、导出） | 不限 |
| **Text** | 无边框文字 | 弱化操作（高级筛选、更多） | 不限 |
| **Danger** | 红色 | 危险操作（删除） | 按需 |

### 6.2 按钮样式规范

| 属性 | 规范值 | 说明 |
|------|--------|------|
| **圆角** | **6px** | 现代友好，保持专业感 |
| 高度 | 32-36px（常规）/ 40px（强调） | 与输入框高度一致 |
| 内边距 | 12px 16px | 左右 16px，上下 12px |
| 字号 | 14px | 正文字号 |
| 字重 | 400（常规）/ 500（强调） | Primary 可用 500 |

### 6.3 页面按钮布局总览

```
┌─────────────────────────────────────────────────────────────┐
│ L1 页面头部                                                  │
│                                                             │
│  患者列表                        ╭──────╮ ╭────────────╮    │
│                                  │ 导出 │ │ + 添加患者 │    │
│                                  ╰──────╯ ╰────────────╯    │
│                                   次要      Primary         │
├─────────────────────────────────────────────────────────────┤
│ L2 筛选区域（背景 #f5f7fa）                                  │
│                                                             │
│  [🔍 搜索] 来源[▼] 状态[▼]       ╭──────╮ ╭──────╮         │
│                                  │ 重置 │ │ 查询 │         │
│                                  ╰──────╯ ╰──────╯         │
│                                   次要      Primary         │
│                                                             │
│  [保险公司▼] [📅 日期范围]                [▽ 高级筛选]      │
│                                            Text 按钮        │
└─────────────────────────────────────────────────────────────┘
```

### 6.4 当前页面按钮问题与修改

| 按钮 | 当前问题 | 修改方案 |
|------|----------|----------|
| **添加新患者** | 文字链接，视觉太弱 | → Primary Button，移到右上角 |
| **搜索** | 灰色，像禁用状态 | → Primary Button（蓝色） |
| **清除** | 样式正确 | ✓ 保持次要按钮样式 |
| **导出** | 在筛选区，尺寸过大 | → 移到页面头部，改为小尺寸次要按钮 |
| **高级筛选** | 居中悬浮 | → 移到筛选区右下角 |
| **所有按钮** | 圆角过小（~2px） | → 统一改为 **6px 圆角** |

## 7. 数据列表区规范（L3）

### 7.1 表格列配置

**推荐列（7-8 列）**：

| 列名 | 宽度 | 对齐 | 说明 |
|------|------|------|------|
| 编号 | 80-100px | 左对齐 | 系统自动生成 |
| 姓名 | 100-120px | 左对齐 | 可点击查看详情 |
| 性别 | 60-80px | 居中 | 男/女 |
| 联系电话 | 130-150px | 左对齐 | 等宽字体 |
| 标签 | 120-150px | 左对齐 | 最多显示 2 个 Tag |
| 来源 | 100-120px | 左对齐 | 渠道来源 |
| 医疗保险 | 100-120px | 左对齐 | 有/无 或具体类型 |
| 操作 | 120-150px | **右对齐** | 查看、编辑 |

**可移除列**：
- "录入人"→ 移入详情页
- 其他低频信息 → 移入详情或展开行

### 7.2 表格样式规范

| 项目 | 规范 |
|------|------|
| **表头背景** | **#fafafa** |
| **表头字重** | **500** |
| 表头字色 | #606266 |
| 表头高度 | 44-48px |
| 固定表头 | 是，滚动时固定 |
| 行高 | ≥ 48px（医疗系统建议 52px） |
| 行边框 | 仅下边框，1px #ebeef5 |
| 斑马纹 | 可选，浅灰 #fafafa |
| 悬停效果 | 背景 #f5f7fa |

### 7.4 操作列规范


## 8. 空状态规范（L3 内部）

### 8.1 核心原则

> **空状态 ≠ 空白**
> 必须提供明确的视觉反馈和下一步引导

### 8.2 空状态结构

```
┌─────────────────────────────────────────────────────────────┐
│ 编号 │ 姓名 │ 性别 │ 联系电话 │ 标签 │ 来源 │医疗保险│ 操作 │ ← 表头保留
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                         👥                                  │
│                    (64px 图标)                              │
│                                                             │
│                   暂无患者数据                               │
│                    (16px 主文案)                            │
│                                                             │
│            请点击「添加患者」开始创建                         │
│               (14px 辅助文案，#909399)                       │
│                                                             │
│                  ╭────────────╮                             │
│                  │ + 添加患者 │                             │
│                  ╰────────────╯                             │
│                  (Primary Button)                           │
│                                                             │
│                  (最小高度 200px)                            │
└─────────────────────────────────────────────────────────────┘
```

### 8.3 空状态规范

| 项目 | 规范 |
|------|------|
| **必须保留表头** | 保持页面结构稳定 |
| 图标 | 64px，主题相关，颜色淡化 |
| 主文案 | 16px，#303133，说明当前状态 |
| 辅助文案 | 14px，#909399，引导下一步 |
| 操作按钮 | Primary Button，与页面头部主操作一致 |
| 最小高度 | 200px，保证视觉平衡 |
| 垂直居中 | 在表格内容区域居中 |

### 8.4 不同场景的空状态文案

| 场景 | 主文案 | 辅助文案 | 操作按钮 |
|------|--------|----------|----------|
| 初始无数据 | 暂无患者数据 | 请点击「添加患者」开始创建 | [+ 添加患者] |
| 筛选无结果 | 未找到匹配的患者 | 请尝试调整筛选条件 | [重置筛选] |
| 搜索无结果 | 未找到"XXX"相关患者 | 请检查输入或尝试其他关键词 | [清空搜索] |

### 8.5 ❌ 禁止项

| 禁止项 | 原因 |
|--------|------|
| 只显示"暂无数据"/"表中没有数据" | 无引导性 |
| 隐藏表头 | 页面结构不稳定 |
| 无操作按钮 | 用户不知道如何添加数据 |
| 空状态区域过小 | 视觉失衡 |

## 12. 响应式规范（新增）

### 12.1 断点定义

| 断点 | 宽度 | 典型设备 |
|------|------|----------|
| xs | < 768px | 手机（不推荐使用） |
| sm | 768px - 1024px | 平板 |
| md | 1024px - 1440px | 小屏笔记本 |
| lg | 1440px - 1920px | 标准桌面 |
| xl | > 1920px | 大屏显示器 |

### 12.2 响应式策略

| 屏幕宽度 | 适配策略 |
|----------|----------|
| **≥ 1440px** | 完整显示所有列和筛选条件 |
| **1024px - 1440px** | 隐藏低优先级列（如"录入人"），筛选条件收起到"更多筛选" |
| **768px - 1024px** | 表格横向滚动，筛选条件折叠为抽屉 |
| **< 768px** | 不推荐，建议使用移动端专用界面 |

### 12.3 列隐藏优先级

当屏幕宽度不足时，按以下顺序隐藏列：

```
优先隐藏 ← ← ← ← ← ← ← ← ← ← ← ← ← 最后隐藏
录入人 → 来源 → 标签 → 医疗保险 → 联系电话 → 性别 → 姓名 → 编号 → 操作
```

| 优先级 | 列 | 说明 |
|--------|-----|------|
| 1（最先隐藏） | 录入人 | 低频查看 |
| 2 | 来源 | 次要信息 |
| 3 | 标签 | 可在详情查看 |
| 4 | 医疗保险 | 次要信息 |
| 5 | 联系电话 | 重要但可收起 |
| 保留 | 编号、姓名、性别、操作 | 核心信息 |

### 12.4 代码示例

```vue
<el-table-column 
  prop="source" 
  label="来源" 
  v-if="screenWidth >= 1200"
/>

<el-table-column 
  prop="inputUser" 
  label="录入人" 
  v-if="screenWidth >= 1440"
/>
```
