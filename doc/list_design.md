# 一、【列表页 HTML / Blade 组件规范】

> 适用于：患者列表、预约列表、收费列表、库存列表、员工列表等
>  核心目标：**信息密度高但不乱、操作明确、长期可扩展**

------

## 1️⃣ 页面结构分层（强制）

列表页 **必须按 4 层结构组织**，禁止随意堆 div。

```
L0 页面容器
 ├─ L1 页面 Header（标题 + 主操作）
 ├─ L2 筛选区（快速筛选 / 高级筛选）
 ├─ L3 数据表格（含表头）
 └─ L4 空状态 / 分页 / 加载态
```

------

## 2️⃣ L1：页面级 Header 规范（必有）

### 结构规范

```
<div class="page-header-l1">
    <h1 class="page-title">患者列表</h1>
    <div class="header-actions">
        [次要操作] [主操作]
    </div>
</div>
```

### 规则

- 左：**页面标题（唯一）**
- 右：**操作按钮（最多 2 个）**
- 主操作：**始终靠最右**
- 高度：`56–64px`

### 示例（标准）

```
<button class="btn btn-default">导出</button>
<button class="btn btn-primary">添加患者</button>
```

🚫 禁止：

- L1 出现筛选项
- L1 出现说明文字
- L1 超过 2 个操作按钮

------

## 3️⃣ L2：筛选区规范

### 设计原则

- **80% 用户只用第一行**
- 高级筛选必须可折叠
- 不允许筛选项散落在表格上方

### 结构规范

```
<div class="filter-area-l2">
    <!-- 高频筛选 -->
    <div class="filter-row">...</div>

    <!-- 高级筛选（默认折叠） -->
    <div id="advancedFilters">...</div>

    <!-- 高级筛选切换 -->
    <button class="advanced-filter-btn">高级筛选</button>
</div>
```

### 高频筛选（第一行）标准内容

- 搜索框（姓名 / 手机）
- 来源 / 状态 / 时间段
- 搜索 / 重置按钮（右对齐）

### 高级筛选（第二行）

- 标签
- 医保 / 公司
- 自定义时间范围

------

## 4️⃣ L3：表格规范

### 表头规则

- 字号：14px
- 字重：500
- 背景：`#fafafa`
- 不使用纯黑文字

### 行规则

- 行高 ≥ 48px
- Hover 必须有背景色
- 操作列右对齐

```
<th>姓名</th>
<th>联系电话</th>
<th class="text-right">操作</th>
```

### 内容规范

- 电话 / 编号 → 等宽字体
- 标签 → Tag 组件，不允许纯文本堆叠
- 操作 ≤ 3 个，超出必须「更多」

------

## 5️⃣ L4：空状态规范

### 原则

- **保留表头**
- 空状态在 tbody 区域或其下
- 必须有“下一步指引”

### 标准结构

```
<div class="empty-state-container">
    <i class="fa fa-users"></i>
    <div>暂无患者数据</div>
    <div>请先添加患者开始管理</div>
    <button class="btn btn-primary">添加患者</button>
</div>
```

🚫 禁止整页空白
🚫 禁止只有“暂无数据”一句话

------

# 二、全系统【按钮语义 + 样式】统一规则

> 这部分**非常值钱**，你可以直接作为前端评审标准。

------

## 1️⃣ 按钮语义分级（强制）

| 类型     | 使用场景           | 组件          |
| -------- | ------------------ | ------------- |
| 主操作   | 新增 / 保存 / 确认 | `btn-primary` |
| 次操作   | 导出 / 编辑 / 查看 | `btn-default` |
| 危险操作 | 删除 / 作废        | `btn-danger`  |
| 文本操作 | 展开 / 更多        | `btn-link`    |

🚫 禁止自造按钮颜色
🚫 禁止同一层级多个 primary

------

## 2️⃣ `<button>` vs `<a>` 使用铁律（重点）

### ✅ 必须用 `<button>`

- 搜索 / 重置
- 新增 / 编辑 / 删除
- 导出
- 折叠 / 展开
- 弹窗触发

### ✅ 只能用 `<a>`

- 页面跳转
- 面包屑
- 详情页导航

👉 **系统中：动作 = button，导航 = a**

------

## 3️⃣ 按钮尺寸统一（你现在已经很接近）

| 级别     | 高度    | 使用位置          |
| -------- | ------- | ----------------- |
| 主按钮   | 36px    | Header / 表单提交 |
| 表格操作 | 28–32px | 行内              |
| 文本按钮 | 自适应  | 高级筛选          |

统一圆角：`6px`

------

## 4️⃣ 状态规范（必须支持）

每一个 button **必须支持以下状态**：

```
default
hover
disabled
loading
```

示例（系统强烈建议）：

```
<button class="btn btn-primary" disabled>
    保存中
</button>
```

------

## 5️⃣ 权限 & 风控预留原则（高级）

> 系统 ≠ 普通后台

- 按钮必须可 **disabled**（不是隐藏）
- 禁用态要有 tooltip 说明
- 不允许只靠 JS 控制可见性
