#### 关键差距说明

**1. 技工单/义齿加工管理（P0 — 口腔诊所刚需）**

技工单是口腔诊所区别于其他医疗机构的核心业务流程，牙医管家和领健均将其作为核心模块。涉及：

```
lab_cases / 技工单
├── 基础信息：患者、医生、义齿类型、加工单位（技工厂）
├── 工艺信息：材料、颜色、咬合记录、特殊要求
├── 流程追踪：送出 → 制作中 → 返回 → 试戴 → 完成
├── 费用管理：加工费、患者收费、利润计算
├── 时间管理：预计交付日期、实际交付日期、超期预警
└── 质量管理：返工记录、质量评价
```

**2. 营销与 CRM（P1 — 商业化竞争力）**

主流竞品（领健 SCRM、牙医管家营销模块）均提供：

```
营销 CRM
├── 患者生命周期管理：新患 → 初诊 → 在治 → 复诊 → 沉睡 → 流失
├── 智能分群：按消费金额、就诊频次、项目类型自动分层
├── 营销活动：优惠券推送、节日营销、裂变活动
├── 渠道追踪：各渠道获客成本、转化率、ROI 分析
├── 沉睡唤醒：自动识别 N 天未就诊患者，触发提醒
└── 微信集成：服务号模板消息、小程序预约、社交分享
```

**3. 患者保险理赔（P1 — 国际市场必需）**

国际市场（Dentrix、CareStack、Open Dental）标配功能：

```
保险理赔管理
├── 保险资格验证：实时查询患者保险状态与覆盖范围
├── 理赔提交：自动生成 CDT 代码、电子提交理赔单
├── EOB 处理：保险公司回款对账、自动过账
├── AR 追踪：应收账款账龄分析、催收工作流
└── 预授权：术前预审批流程
```

**4. 中国市场合规缺口（P0 — 商业化前置条件）**

原系统技术架构成熟，但功能逻辑偏国际 SaaS 模型。中国市场商业落地需补齐以下核心能力：

- **医保结算对接** — 中国口腔门诊刚需，没有医保结算诊所不会买单。需对接各地医保局接口，支持医保/自费混合支付。
- **电子病历合规** — 《电子病历基本规范》要求签名锁定、修改留痕、版本管理、最低保存 15 年。
- **财税合规** — 金税四期已全面推行，必须支持电子发票生成、红冲、作废、税率管理、收款与发票对账。
- **数据安全等保** — 医疗 SaaS 等保二级起步，需 AES 加密敏感字段、审计日志不可删除、异地备份、私有化部署支持。
- **器械消毒追溯** — 口腔门诊监管检查重点，需器械包登记、灭菌批次号、关联患者、过期提醒。
- **收费合规** — 物价局监管要求收费透明，需价格锁定、变更审批流、收费公示清单导出。

> 目标：从"技术完善型系统"升级为"可在中国市场商业落地的医疗 SaaS 平台"。

#### 对标总结

**路线图覆盖率约 60-65%**，主要差距集中在：

1. **AI 能力（差距最大）**：e看牙已形成 AI 影像诊断 + 语音病历 + 智能推荐产品矩阵，路线图完全未涉及 → 新增 5.4 节
2. **BI 数据平台（差距大）**：e看牙"有数"是独立 BI 平台（三档版本），5.1 仅覆盖基础报表 → 扩充 5.1
3. **SCRM 深度（差距中）**：e看牙"有客"是完整 CDP + MA 平台，4.2 Phase A/B 覆盖方向但深度不足 → 扩充 4.2
4. **专科模块（差距中）**：正畸/种植/儿牙专科管理是差异化卖点 → 新增 5.5 节
5. **供应链/商城（差距中）**：B2B 耗材采购商城是生态闭环能力 → 远期 P3
6. **智能硬件（差距低）**：智能工牌、叫号屏属锦上添花 → 暂不纳入

> **结论**：路线图 P0/P1 优先级排序（医保→财税→支付→小程序→SCRM）与 e看牙核心能力高度吻合，方向正确。主要补充 AI 规划、BI 平台深度和专科扩展模块。

## 商业化定价建议

基于口腔诊所管理 SaaS 市场：

| 版本 | 目标客户 | 核心功能 | 限制 |
|---|---|---|---|
| 免费版 | 个体诊所试用 | 患者 + 预约 + 基础账单 | 1 用户、200 患者 |
| 标准版 | 单店诊所 | 全功能 + 库存 + 报表 | 5 用户、1 门店 |
| 专业版 | 连锁诊所 | 多门店 + 会员 + 绩效 + API | 20 用户、5 门店 |
| 企业版 | 大型连锁/DSO | 自定义域名 + 数据分析 + 专属支持 | 不限 |

**建议首先推出标准版**，项目当前功能已覆盖单店诊所核心需求，阶段一加固完成后即可开始小范围收费试运营，用真实客户反馈驱动后续迭代优先级。
